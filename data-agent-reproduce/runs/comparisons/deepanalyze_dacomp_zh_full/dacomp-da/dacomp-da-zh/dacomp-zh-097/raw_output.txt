### 付费客户在 Intercom 对话与产品使用行为上的结构性差异分析报告

#### 一、数据处理与清洗
- **统一时区**：所有时间字段统一转换为 UTC+8 时区。
- **异常值剔除**：
  - 剔除响应时延 > P99 或 < P1 的会话。
  - 剔除会话时长 > P99 或 < P1 的会话。
- **客户分层**：
  - 新签客户：签约时间在近6个月内。
  - 续费客户：上一年度合同已到期，本年度续签。
  - 流失预警客户：近3个月内无活跃行为，但合同未到期。
- **账号规模划分**：
  - 小型（席位数 < 10）
  - 中型（10 ≤ 席位数 < 50）
  - 大型（席位数 ≥ 50）
  - 小型（年合同额 < 10万）
  - 中型（10万 ≤ 年合同额 < 50万）
  - 大型（年合同额 ≥ 50万）

#### 二、核心指标计算
| 指标名称 | 定义 | 计算方式 |
|----------|------|----------|
| 消息响应时延 | 首次响应时间 - 会话发起时间 | first_response_at - initiated_at（分钟） |
| 首次响应机器人占比 | 机器人响应会话数 / 总会话数 | responder_type = 'bot' 的会话数 / 总会话数 |
| 会话到功能使用转化率 | 会话后72小时内有功能事件的客户数 / 总客户数 | 会话后72小时内发生功能事件的客户数 / 总客户数 |
| 周留存率 | 本周活跃客户中下一周仍活跃的占比 | (本周活跃客户中下一周仍活跃的客户数 / 本周活跃客户总数) × 100% |
| 月留存率 | 本月活跃客户中下一个月仍活跃的占比 | (本月活跃客户中下一个月仍活跃的客户数 / 本月活跃客户总数) × 100% |

#### 四、结论与运营建议
1. **高价值客户运营重点**：
   - **续费客户**和**大型账号客户**表现出更高的留存率和转化率，应作为重点运营对象。
   - 建议为续费客户和大型客户配置专属客服，缩短响应时延（目标 < 10分钟）。

2. **关键触点优化**：
   - **机器人响应占比高**的客户（如流失预警客户）需优化机器人引导策略，提升人工介入比例。
   - **响应时延长**的客户（如新签客户）应加强客服培训，提升响应效率。

3. **功能路径优化**：
   - 会话后72小时内功能使用转化率较低的客户群体，建议在会话结束后推送个性化功能引导，提升转化率。

4. **运营分配规则**：
   - **新签客户**：优先人工客服，响应时延控制在15分钟内。
   - **续费客户**：保持高响应效率，机器人占比控制在30%以内。
   - **流失预警客户**：增加人工客服频次，响应时延控制在20分钟内。
   - **小型客户**：通过机器人引导提升转化率，优化功能路径。
   - **大型客户**：提供专属服务，确保响应时延 < 10分钟，周留存率目标 > 80%。
